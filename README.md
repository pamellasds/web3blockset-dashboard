# Web3BlockSet: GitHub Data Collection Pipeline

Dataset for the paper: **"Web3BlockSet: A Dataset for Empirical Research in Blockchain-Oriented Software Engineering"**

---

## Prerequisites

```bash
# Python 3.8+
pip install -r requirements.txt
```

Or manually:
```bash
pip install pandas nltk pyyaml requests PyGithub tqdm
```

### GitHub Token Setup

```python
# In each collector script, replace:
GITHUB_TOKEN = "ghp_your_token_here"
```

---

## Execution Flow

### Step 1: Collect Repository Metadata
Collects Web3 repository metadata via GitHub API.

```bash
python 1_repo_info_collector.py
```

**Input**: `additional_files/web3_organizations.json`  
**Output**: 
- `additional_files/repositories.csv` (all repos)
- `additional_files/repositories.yaml` (filtered: >300 stars, >1 issue) - It was processed to be used in the next step.

---

### Step 2: Collect Issues, PRs, and Commits
Extracts issues, PRs, and commits from filtered repositories via GitHub API

#### Provider Repositories (Official)

```bash
python 2_prs_issues_commits_collector.py --providers
```

**Input**: `additional_files/repositories.yaml`  
**Output**: `prs_issues_commits_collected/providers/data/`

#### Community Repositories (Search)

```bash
python 2_prs_issues_commits_collector.py --communities
```

**Input**: `additional_files/keywords.json`  
**Output**: `prs_issues_commits_collected/communities/data/`

---

### Step 3: Combine and Filter Bot Content
Combines data and removes bot-generated content.

```bash
python 3_repo_comments_combiner.py
```

**Input**: `prs_issues_commits_collected/{providers,communities}/data/`  
**Output**: 
- `repo_comments_combiner_collected/providers_combined.csv`
- `repo_comments_combiner_collected/communities_combined.csv`

---

### Step 4: Preprocess Text Data
Preprocesses text (cleaning, normalization, stemming).

```bash
python 4_preprocessor.py
```

**Input**: Combined CSV files from Step 3  
**Output**: `web3blockset/issues_prs.csv` (final dataset)
---

## Configuration Files

### `web3_organizations.json`

Simple array of GitHub organization names:

```json
[
  "ethereum",
  "OpenZeppelin",
  "Uniswap",
  ...
]
```

**Source**: Extracted from [Awesome Web3](https://github.com/ahmet/awesome-web3) list.  
---

### `repositories.yaml` (Generated by Step 1)

Contains filtered repository URLs that meet quality criteria (Repositories with >300 stars AND >1 open issue):

```yaml
repositories:
  - https://github.com/0xsequence/erc-1155
  ...
```

---

### `keywords.json`

Search patterns for community dataset collection:

```json
{
  "mandatory_terms": ["blockchain"],
  "search_patterns": {
    "0xsequence": [
      "0xsequence",
      "https://github.com/0xsequence/erc-1155",
      "sequence.js"
    ],
    ...
  }
}
```

**Logic**:
- **`mandatory_terms`**: Must appear in every matched issue/PR (filters out non-blockchain content)
- **`search_patterns`**: Keywords organized by platform/ecosystem
- **Keyword sources**: 
  - Repository names from `repositories.yaml` (e.g., "hardhat" from hardhat-org/hardhat)
  - Library/package names mentioned in repos
  - Common variants and related tools

## Dataset Schema

### `issues_prs.csv` (391,596 records)

| Column | Type | Description |
|--------|------|-------------|
| `repository` | string | Repository name |
| `owner` | string | Organization/user |
| `issue_id` | int | GitHub issue ID |
| `issue_number` | int | Issue number in repo |
| `issue_title` | string | Title text |
| `issue_body` | string | Body text |
| `state` | string | open/closed |
| `created_at` | datetime | Creation timestamp |
| `updated_at` | datetime | Last update |
| `closed_at` | datetime | Close timestamp (null if open) |
| `author` | string | Issue author username |
| `author_id` | int | Author GitHub ID |
| `locked` | boolean | Is discussion locked? |
| `comments_count` | int | Total comment count |
| `commits_count` | int | Number of commits (PRs only) |
| `labels` | string | Comma-separated labels |
| `type` | string | Issue / Pull Request |
| `concatenated_comments` | string | All comments joined with ' | ' |
| `comment_authors` | string | Semicolon-separated comment authors |
| `comments_count_filtered` | int | Comment count after bot filtering |
| `repository_category` | string | Repository classification (validated) |
| `year` | int | Year created (extracted from created_at) |
| `raw_text` | string | Title + body + comments (original) |

## Repository Categorization (LLM-Assisted)
Repository classification was performed using LLM assistance a priori:

**Process**:
1. Export filtered repositories from Step 1 to CSV
2. Submit to LLM with prompt:

```
Given this CSV of GitHub repositories, classify each repository by its nature/purpose.
Consider the repository name, description, and organization context.
Return the same CSV with an additional 'repository_category_claude' column.
```

2. **Human Validation**: An experienced BOSE researcher reviews and adjusts the LLM suggestions, creating the `repository_category` column

3. **Cross-Validation**: Four additional BOSE researchers validate the classifications

4. **Result**: Final `repository_category` column added to `repositories.csv` and propagated to the dataset

NOTE: we finalized the repositories.csv using `repository_category` after human validation.
